# 骨科康复智能系统  
## 代码与数据接口统一规范（第一阶段：模拟验证）

---

## 1. 目标说明

本项目当前阶段的首要目标是 **“流程跑通”**，而非模型精度。

为确保未来可无缝接入真实临床数据，现阶段需统一：
- 数据存放位置
- 数据字段结构
- 数据加载接口形式

---

## 2. 数据存放规则（强制）

所有模块生成的中间或最终数据，必须存放在项目根目录下的 `data/` 文件夹中。

```
data/
├── icf/    # ICF 评估与结局预测模块
├── gait/   # 步态分析与异常检测模块
├── sensor/ # IMU 动作识别模块
└── fusion/ # 多模态融合与决策模块
```

---

## 3. 各模块输出数据格式规范（CSV）

### 3.1 ICF 模块（data/icf/）

CSV 文件至少包含以下字段：

| 字段名 | 含义 |
|------|------|
| patient_id | 患者唯一编号 |
| time_step | 时间序列索引 |
| rehab_phase | 康复阶段（early / mid / late） |
| icf_total | ICF 总分 |
| rom | 关节活动度 |
| vas | 疼痛评分 |

---

### 3.2 步态模块（data/gait/）

| 字段名 | 含义 |
|------|------|
| patient_id | 患者编号 |
| gait_abnormal_prob | 步态异常概率（0-1） |

---

### 3.3 IMU 动作识别模块（data/sensor/）

| 字段名 | 含义 |
|------|------|
| patient_id | 患者编号 |
| action_type | 动作类型 |
| quality_score | 动作质量评分（模拟） |

---

## 4. 数据加载接口规范（关键）

每个模块 **必须提供一个独立的数据加载函数或类**，满足以下要求：

- 输入：文件路径（字符串）
- 输出：结构化数据（DataFrame / dict / Tensor）
